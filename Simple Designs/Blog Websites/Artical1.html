Syntax vs Semantics

Posted by Mohammad Ahmad on Sunday, August 25st 2024

At the beginning of your programming journey, you will discover that there are many programming languages to learn. During this process, you will frequently encounter two important terms: "syntax" and "semantics." These two concepts form the core of any programming language, and understanding them correctly is essential for progressing effectively in learning to code. In this article, we will explore the difference between "syntax" and "semantics" and highlight their importance in programming languages.

Syntax

When writing code, there is a specific set of words that must follow a certain order to ensure the code is valid. This is where syntax comes into play, which is a set of rules and guidelines that dictate how to write statements in a programming language. It ensures that the statements follow the rules regarding the words used, their arrangement, and so on.

Let's take the following sentence as an example:
Ate today apple went school.
This sentence is syntactically invalid because the order is illogical, and some words are missing. If we fix it, we get:
I ate an apple today before going to school.
In this corrected sentence, the order is correct, and the syntax follows proper grammatical rules.

Key aspects of syntax:

  • Grammar rules: Syntax involves formal grammar rules that define the order of elements in code and their hierarchical relationship.
  • Keywords and symbols: Programming languages use specific keywords and symbols that must be used correctly.

Semantics

Semantics focuses on the meaning and logic of code. Even when syntax is valid, the semantics determine whether the code’s behavior is meaningful and logical. Semantics determines what the statement means, what actions it will perform, and the order in which the statement will be executed. It deals with understanding the statements to achieve the desired result.

Sameer is a single married man.
